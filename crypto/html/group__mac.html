MAC API#

A Message Authentication Code (MAC) is a cryptographic technique used to verify the authenticity and integrity of a message, ensuring that it has not been tampered during transmission.

Data Structures#

struct alc_hmac_info_t#

Stores details of HMAC.

Param hmac_digest:

Store info of digest used for HMAC

struct alc_cmac_info_t#

Stores details of CMAC.

Param cmac_cipher:

Store info of cipher used for CMAC

struct alc_poly1305_info_t#

Stores details for Poly1305.

struct alc_mac_info_t#

Stores details of CMAC.

Note

Supported MAC algorithms HMAC, CMAC, POLY1305

Param mi_type:

Store Type of MAC to be used

Param mi_algoinfo:

A Union of alc_hmac_info_t, alc_cmac_info_t

Param mi_keyinfo:

Store key info

struct alc_mac_handle_t#

Handle for maintaining session.

Param ch_context:

pointer to the context of the mac

Functions#

alc_error_t alcp_mac_supported(const alc_mac_info_p pcMacInfo)#

Allows to check if a given algorithm is supported or not.

This API needs to be called before any other API is called to know if mac that is being request is supported or not

Note

alcp_mac_supported() is called first to know if the given configuration is valid.

Parameters:

pcMacInfo – [in] Description of the requested mac session

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_error_str needs to be called to know about error occurred

Uint64 alcp_mac_context_size(const alc_mac_info_p pMacInfo)#

Gets the size of the context for a session described by pMacInfo.

This API can be called before alcp_mac_request to identify the memory to be allocated for context

Parameters:

pMacInfo – [in] Description of the requested MAC session

Returns:

Size of Context

alc_error_t alcp_mac_request(alc_mac_handle_p pMacHandle, const alc_mac_info_p pMacInfo)#

Allows caller to request for a MAC as described by pMacInfo.

This API must be called before making any other API call

Note

Error needs to be checked after each call, valid only if alcp_is_error (ret) is false

Parameters:

  • pMacInfo – [in] Description of the MAC session

  • pMacHandle – [out] Library populated session handle for future mac operations.

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_update(alc_mac_handle_p pMacHandle, const Uint8 *buff, Uint64 size)#

Allows caller to update MAC with chunk of data to be authenticated.

This API is called to update data to be authenticated. So should be called after alcp_mac_request and before the end of session call, alcp_mac_finish

Note

Error needs to be checked for each call, valid only if alcp_is_error (ret) is false

Parameters:

  • pMacHandle – [in] Session handle for future MAC operation

  • buff – [in] The chunk of the message to be updated

  • size – [in] Length of input buffer in bytes

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_finalize(alc_mac_handle_p pMacHandle, const Uint8 *buff, Uint64 size)#

Allows caller to finalize MAC with final chunk of data to be authenticated.

This API is called to finalize mac so should be called after alcp_mac_request and before alcp_mac_finish

Note

  • Error needs to be checked for each call, valid only if alcp_is_error (ret) is false.

  • It is expected that application calls alcp_mac_copy() after calling this function as the contents of the session is not guaranteed to persist after alcp_mac_finish()

Parameters:

  • pMacHandle – [in] Session handle for future MAC operation

  • buff – [in] The last chunk of the message to be updated

  • size – [in] Length of input buffer in bytes.

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_copy(alc_mac_handle_p pMacHandle, Uint8 *buff, Uint64 size)#

Allows caller to copy MAC to the buffer.

This API is called to copy mac, so should be called after alcp_mac_request and before alcp_mac_finish

Note

Error needs to be checked for each call, valid only if alcp_is_error (ret) is false

Parameters:

  • pMacHandle – [in] Session handle for future MAC operation

  • buff – [out] Destination buffer to which mac will be copied

  • size – [in] Length of output buffer in bytes

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_finish(alc_mac_handle_p pMacHandle)#

Free resources that was allotted by alcp_mac_request.

This API is called to free resources so should be called to free the session

Note

alcp_mac_request() should be called first to allocate the Handler.

Parameters:

pMacHandle – [in] Session handle used for the MAC operations

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_reset(alc_mac_handle_p pMacHandle)#

resets the data given to it during alcp_mac_update or alcp_mac_finalize

This API is called to reset data so should be called after alcp_mac_request and before alcp_mac_finish

Parameters:

pMacHandle – [in] Session handle for future MAC operation

Returns:

Error Code for the API called. If alc_error_t is not ALC_ERROR_NONE then alcp_mac_error or alcp_error_str needs to be called to know about error occurred

alc_error_t alcp_mac_error(alc_mac_handle_p pMacHandle, Uint8 *pBuff, Uint64 size)#

Get the error string for errors occurring in MAC operations.

This API is called to get the error string. It should be called after alcp_mac_request and before alcp_mac_finish

Parameters:

  • pMacHandle – [in] Session handle for MAC operation

  • pBuff – [out] Destination Buffer to which Error String will be copied

  • size – [in] Length of the Buffer.

Returns:

alc_error_t Error code to validate the Handle
